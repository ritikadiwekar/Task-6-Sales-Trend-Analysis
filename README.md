# 📈 Sales Trend Analysis Using SQL

## 🚀 Project Summary

In this project, I analyzed sales data using SQL to identify monthly revenue trends, order volume patterns, and product performance.

The goal was to transform raw sales data into meaningful business insights using SQL aggregation functions and analytical queries.

---

## 🛠 Tool Used

- MySQL Workbench

---

## 📊 Business Questions Solved

### Revenue Analysis
- What is the monthly revenue?
- Which month generated the highest revenue?
- Which month generated the lowest revenue?

### Order Analysis
- What is the monthly order volume?
- How many orders were placed for each product?

### Product Performance
- Which products generated the highest revenue?
- What are the top 3 products by revenue?
- What percentage of total revenue does each product contribute?

### Advanced Analysis
- Average Order Value by Month
- Revenue Above Threshold using HAVING Clause
- Revenue Contribution using Subqueries

---

## 🧠 SQL Concepts Applied

```sql
SUM()
COUNT()
COUNT(DISTINCT)
AVG()
GROUP BY
ORDER BY
LIMIT
WHERE
HAVING
EXTRACT()
ROUND()
SUBQUERY
```

---

## 📌 Key Insights

✅ Revenue fluctuated across different months.

✅ Certain products consistently generated higher revenue.

✅ Monthly order volume helped identify sales patterns.

✅ Revenue contribution analysis highlighted the most valuable products.

✅ SQL aggregations helped convert raw transactional data into actionable business insights.

---

## 🏁 Conclusion

This project demonstrates how SQL can be used to analyze sales performance and support business decision-making. By leveraging aggregation functions, filtering techniques, and subqueries, meaningful insights were extracted from sales data.
